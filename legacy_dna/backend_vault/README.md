# Backend Code Vault (The Hydration Engine)

هذا المجلد يحتوي على النسخ الأصلية (JS/Node.js) للمحرك القديم الذي كان مسؤولاً عن تطبيق الـ **Vertical Language Translation** والـ **Smart Hydration**.
تم حفظ هذه الملفات كمرجع برمجي (DNA) لكي نستلهم منها المنطق المعماري عند بناء محرك الـ C++ أو الـ SQL Views في النظام الجديد.

## الملفات المحفوظة:

### 1. `smartSchema.js`
- **الوظيفة:** هو المخ (Brain) الذي كان يقرأ الـ Schema الأصلية.
- **العبقرية:** يقوم بتحليل هيكل الجداول، يكتشف تلقائياً أي جدول ينتهي بـ `_lang`، ويبني خريطة بالعلاقات (Foreign Keys) والجداول القابلة للترجمة.

### 2. `i18nLoader.js`
- **الوظيفة:** طبقة سحب الترجمات (Translation Fetcher).
- **العبقرية:** يحتوي على دالة `attachTranslationsToRows` التي كانت تأخذ صفوف الجدول الأساسي (Base rows)، وتبحث عن ما يقابلها في جدول اللغات، وتقوم بحقن الترجمة في كائن فرعي `i18n.lang.ar` ليصبح الصف مسطحاً برمجياً.

### 3. `hydrator.js`
- **الوظيفة:** الساحر الأكبر (The Grand Hydrator).
- **العبقرية:** يقرأ خريطة العلاقات من `smartSchema.js`. إذا وجد أن سجل المريض يحتوي على `city_id`، يذهب إلى جدول المدن، يجلب بيانات المدينة، ثم يمرر بيانات المدينة إلى `i18nLoader.js` لجلب ترجمتها، وأخيراً يرسل للواجهة كائناً متداخلاً (Nested Object) مسطحاً ومترجماً بالكامل.
  
---
**الهدف من هذا المخزن:** 
عندما نقوم بكتابة `compiler.py` لإنشاء الـ SQL Views التلقائية في النظام الجديد، يجب أن ننظر في هذا المنطق ونحوله من (JS Runtime) إلى (SQL DDL Build time).
